## 1. A marca em uma frase

**Hotlist é o checklist do Diecast Club** — um clube digital de colecionadores de diecast, começando por Hot Wheels e crescendo para abraçar Matchbox, Majorette, Tomica, Maisto e o que mais a comunidade colecionar.

- **Hotlist** é o produto (o app, a ferramenta, a "lista quente" do colecionador).
- **Diecast Club** é o clube (a casa, a comunidade, o universo da marca).

Os dois andam juntos como **Hotlist — Diecast Club** em assinaturas oficiais.

---

## 2. Essência

### Propósito
Dar ao colecionador de diecast a forma mais simples, rápida e bonita de organizar, registrar e valorizar sua coleção — independentemente de qual marca ele coleciona.

### Visão
Ser o ponto de encontro digital dos colecionadores de diecast no Brasil. Começamos com Hot Wheels porque é onde está a maior comunidade. Crescemos abraçando as outras marcas porque colecionador raiz não coleciona só uma.

### Missão
Construir uma plataforma sólida, sem propaganda, sem fricção e sem dono externo — feita por quem coleciona, para quem coleciona.

### Posicionamento
**Hotlist — Diecast Club** é o **clube e ferramenta de colecionadores de diecast**. Não é loja, não é marketplace, não é fórum, não é rede social. É o lugar onde sua coleção mora, organizada e respeitada.

### Arquitetura de marca

- **Hoje:** Hotlist suporta Hot Wheels.
- **Em breve:** o app é modelado para receber outras marcas (Matchbox, Majorette, Tomica, Maisto e adjacentes) sem virar uma colcha de retalhos.
- **Princípio guia:** o nome do produto (Hotlist) e o nome do clube (Diecast Club) são marca-agnósticos. Em nenhum lugar da identidade aparece "Hot Wheels" como parte do nome ou do logo.

---

## 3. Personalidade

A marca se comporta como **o curador do clube**: organiza a coleção sem se intrometer, conhece todas as marcas sem preferir nenhuma, e respeita o hobby como ele é.

- **Direto** — vai ao ponto.
- **Curador** — sabe a diferença entre um STH, um Premium da Matchbox e um Tomica Limited. Trata cada um com o respeito que merece.
- **Marca-agnóstico** — não puxa para nenhum lado.
- **Discreto** — o protagonista é a coleção, não o app.
- **Confiável** — seus dados ficam onde devem ficar.

### Arquétipo
**O Curador.** Não é vendedor, não é animador, não é especialista que dá lição. É o cara que organiza a coleção do clube e te entrega a lista impecável quando você pede.

## 4. Tom de voz

| Somos | Não somos |
|---|---|
| Claros e curtos | Verbosos, formais, corporativos |
| Brasileiros e coloquiais | Robóticos ou traduzidos do inglês |
| Específicos do hobby (mainline, STH, RLC, carded, loose, premium, Tomica Limited) | Genéricos como "miniaturas colecionáveis" |
| Pragmáticos | Sentimentais ou exagerados |
| Marca-agnósticos | "Hot Wheels é melhor" / "Matchbox é coisa de criança" |
| Honestos sobre erros | Defensivos |

### Princípios de escrita

1. **Frase curta vence frase longa.** Se cabe em uma linha, fica em uma linha.
2. **Verbo no infinitivo para ações.** "Adicionar item", "Exportar backup", "Escanear código".
3. **Sem "por favor" e "obrigado" em microcopy.** Botões são botões.
4. **Erros sem culpa.** Diga o que aconteceu e o que fazer.
5. **Vocabulário do hobby fica em inglês quando original.** "Mainline", "carded", "loose", "Treasure Hunt", "premium" — não traduza.
6. **Nunca favoreça uma marca.** Se a frase só funciona pra Hot Wheels, reescreva pra funcionar pra qualquer diecast.

### Exemplos

| Contexto | ✅ Sim | ❌ Não |
|---|---|---|
| Onboarding | "Sua coleção de diecast, organizada." | "Seu app de Hot Wheels favorito!" |
| Empty state | "Nenhum carrinho ainda. Adicione o primeiro." | "Você ainda não cadastrou itens." |
| Erro de import | "Arquivo inválido. Use um .json exportado pelo app." | "Ocorreu um erro ao processar o arquivo." |
| Confirmação | "Coleção criada" | "Sua nova coleção foi criada com sucesso!" |
| CTA | "Entrar com Google" | "Clique aqui para fazer login" |

## 6. Logotipo (arquivos oficiais)

O logo do Hotlist — Diecast Club é um **lockup completo**: o wordmark "HOTLIST" inscrito em uma elipse dupla concêntrica, com o descritor "DIECAST CLUB" em Hanken Grotesk Italic abaixo.

**Especificações técnicas:**
- **viewBox:** `0 0 623.29 476.26` (proporção ~1.31:1)
- **Tipografia interna:** Hanken Grotesk Italic, 400, 40px no SVG
- **Stroke:** 2px na elipse e nos caracteres (preservar ao redimensionar)
- **Versões disponíveis:** `public/logo-white.svg` (fundo escuro) e `public/logo-black.svg` (fundo claro)

### 6.1. Versão branca — `public/logo-white.svg`

Para uso sobre fundos escuros. Todos os elementos em `#fff`.

### 6.2. Versão preta — `public/logo-black.svg`

Para uso sobre fundos claros (branco ou cinza claríssimo). Todos os elementos em `#000`.

### 6.3. Versão adaptativa (`currentColor`) — recomendada no app

Para uso em React/HTML quando o logo deve herdar a cor do CSS pai automaticamente.

```tsx
// src/components/Logo.tsx
export function Logo({ className }: { className?: string }) {
  return (
    <svg
      xmlns="http://www.w3.org/2000/svg"
      viewBox="0 0 623.29 476.26"
      className={className}
      style={{ color: 'inherit' }}
      aria-label="Hotlist — Diecast Club"
    >
      {/* Usar currentColor em fill e stroke */}
    </svg>
  )
}

// Uso:
<Logo className="h-8 w-auto text-zinc-900 dark:text-white" />
```

### 6.4. Como o logo é construído

O logo tem **três elementos**:

1. **Elipse externa dupla** (outline 2px, sem preenchimento) — a "pista" da marca.
2. **Wordmark "HOTLIST"** — letras geométricas pesadas, centradas na elipse. Wordmark customizado, não é fonte.
3. **Descritor "DIECAST CLUB"** em **Hanken Grotesk Italic 400**, com letter-spacing variável (0 a 0.03em), posicionado abaixo do wordmark dentro da elipse.

## 7. Identidade visual

### 7.1. Base técnica: shadcn/ui + Tailwind CSS v3

A identidade visual é construída sobre o **[shadcn/ui](https://ui.shadcn.com/)** com **Tailwind CSS v3**.

**Configuração atual:**
- `baseColor`: **zinc** (neutro quase sem saturação).
- `cssVariables`: **true** — valores em HSL.
- `style`: **new-york**.

### 7.2. Princípios visuais

1. **Clean/minimalista.** Fundo branco, tipografia faz o trabalho.
2. **Monocromático.** A marca vive em preto, branco e cinzas zinc.
3. **Marca-agnóstica.** Nada na identidade pode parecer "de uma marca específica" de diecast.
4. **Semântica acima de estética.** Use `bg-primary`, não `bg-zinc-900`.
5. **Cores funcionais são exceção.** Só para raridade, posse confirmada e erros.

### 7.3. Tokens semânticos (shadcn/ui)

| Token | Propósito |
|---|---|
| `--background` | Fundo principal da aplicação |
| `--foreground` | Cor padrão de texto |
| `--card` / `--card-foreground` | Fundo e texto de cards |
| `--popover` / `--popover-foreground` | Dropdowns, popovers |
| `--primary` / `--primary-foreground` | Ação principal, CTA |
| `--secondary` / `--secondary-foreground` | Ação secundária |
| `--muted` / `--muted-foreground` | Texto secundário, labels |
| `--accent` / `--accent-foreground` | Hover sutil, estado ativo |
| `--destructive` / `--destructive-foreground` | Erros e ações destrutivas |
| `--border` / `--input` / `--ring` | Bordas, campos, foco |
| `--radius` | Raio base (`0.5rem`) |

### 7.5. Cores funcionais do Hotlist

Três cores com significado específico no app, **nunca usadas como decoração de marca**:

| Token / classe | Significado | Uso |
|---|---|---|
| `bg-emerald-500` | Posse confirmada | Indicador verde no item da lista/grid |
| `text-destructive` | Ação destrutiva | Botão excluir, erro de formulário |
| Cor dinâmica via `meta.badgeBg` | Raridade da linha | Badge RLC, STH, Premium, TH — cor vem dos dados |

> ⚠️ As cores de raridade (RLC, STH, etc.) são **inline styles dinâmicos** — não tokens Tailwind — porque dependem dos dados de cada linha. Preservar esse padrão ao refatorar.

## 8. Componentes — uso prático

Toda a UI é construída com componentes shadcn/ui em `src/components/ui/`.

| Componente | Uso no Hotlist |
|---|---|
| `Button` | CTA (`default`), secundárias (`outline`, `ghost`), destrutivas (`destructive`) |
| `Input` | Busca, formulários de item, preço |
| `Select` | Seleção de linha, embalagem, coleção |
| `Sheet` | Painel de detalhe do item (`side="right"`), adicionar item (`side="bottom"`) |
| `ScrollArea` | Corpo scrollável dos painéis |
| `Badge` | Linha do item (mainline, STH, Premium, RLC, TH) |
| `Avatar` + `AvatarImage` | Foto do usuário no header |
| `DropdownMenu` | Menu do usuário (nome, email, sair) |
| `Separator` | Divisórias em menus e seções |
| `Sonner` | Toasts de confirmação *(roadmap)* |
| `Dialog` | Modal de galeria de fotos |

### Princípios

1. **Não estilize por cima.** Ajuste via token (`--primary`, `--radius`), não via override de classe.
2. **Variants antes de classes.** Use `<Button variant="outline">`, não `<button className="border ...">`.
3. **Mobile-first sempre.** `Sheet` antes de `Dialog` em fluxos mobile.
4. **Inline styles apenas para cores dinâmicas** — badges de raridade com `style={{ background: meta.badgeBg }}`.

## 9. Diretrizes de uso do logotipo

### 9.1. Versões oficiais

- **`public/logo-black.svg`** — fundos claros (uso padrão no app).
- **`public/logo-white.svg`** — fundos escuros.
- **Versão `currentColor`** — uso interno via React para herdar cor do tema.

### 9.2. Lockup

O logo é sempre o lockup completo. Não existe versão "só HOTLIST" ou "só elipse" sem aprovação. Use o SVG inteiro como unidade.

### 9.3. Área de respiro

Mínimo = **altura da letra "H"** do wordmark (~76 unidades no viewBox). Em CSS ≈ 12% da largura do logo em cada lado.

### 9.4. Tamanho mínimo

- **Tela:** 80px de largura. Abaixo disso, "DIECAST CLUB" fica ilegível.
- **Impresso:** 25mm de largura.
- **Favicon/ícone PWA:** não usar o lockup completo. O ícone atual (`public/icons/`) é um símbolo derivado gerado a partir do SVG.

### 9.5. O que nunca fazer

1. ❌ Recolorir com cor diferente de preto, branco ou `currentColor`.
2. ❌ Aplicar contorno, sombra, brilho ou bisotê.
3. ❌ Distorcer (manter proporção 1.31:1 do viewBox).
4. ❌ Rotacionar fora de 0°.
5. ❌ Usar sobre fundo com contraste insuficiente.
6. ❌ Separar elementos (elipse sem texto, ou texto sem elipse) sem aprovação.
7. ❌ Substituir a fonte do "DIECAST CLUB". Sempre Hanken Grotesk Italic 400.
8. ❌ Alterar o stroke da elipse (sempre 2px no viewBox, escala proporcionalmente).
9. ❌ Combinar com outro logo no mesmo lockup sem aprovação.
10. ❌ Usar como pattern repetido ou textura.
11. ❌ Animar de forma que descaracterize a forma.

### 9.6. Isenção legal

**Hotlist — Diecast Club não é afiliado, endossado ou patrocinado por Mattel (Hot Wheels, Matchbox), Majorette, Takara Tomy (Tomica), Maisto ou qualquer fabricante de diecast.**

Em material público, incluir:

> *Hot Wheels®, Matchbox®, Majorette®, Tomica® e Maisto® são marcas registradas de seus respectivos proprietários. Hotlist — Diecast Club é um projeto independente, sem vínculo oficial com qualquer fabricante.*

## 10. Stack técnica atual

O app está em produção em **[hotlist.com.br](https://hotlist.com.br)**.

### Frontend
- **React 18** + **TypeScript** + **Vite 7**
- **Tailwind CSS v3** + **shadcn/ui** (new-york, zinc)
- **PWA** via `vite-plugin-pwa` (Workbox)
- **Lucide React** para ícones

### Backend / infra
- **Firebase Authentication** — Google Sign-In
- **Firebase Firestore** — dados do usuário + catálogo colaborativo de preços
- **GitHub Pages** — hospedagem (branch `main`, pasta `docs/`)
- **Domínio:** `hotlist.com.br` com HTTPS via Let's Encrypt

### Estrutura de pastas relevante

```
src/
  components/
    ui/          # Componentes shadcn/ui
    Header.tsx   # Toolbar + abas de navegação + menu do usuário
    LoginScreen.tsx
    ItemDetail.tsx   # Sheet lateral (detalhes + preços)
    AddItemSheet.tsx # Sheet inferior (adicionar item)
    Stats.tsx
    GridView.tsx
    SeriesList.tsx
    Editor/      # Gerenciamento de coleções + backup
  hooks/
    useAuth.ts
    useHotlist.ts
  lib/
    firebase.ts
    catalog.ts   # Preços colaborativos
  styles/
    global.css   # Tailwind directives + shadcn tokens
public/
  logo-black.svg   # Logo para fundos claros
  logo-white.svg   # Logo para fundos escuros
  icons/           # Ícones PWA (192, 512)
  CNAME            # hotlist.com.br
```

## 11. Checklist antes de publicar qualquer arte

- [ ] Versão correta do logo (preta em claro, branca em escuro)?
- [ ] Respiro de pelo menos 1× a altura da letra H do wordmark?
- [ ] Logo no tamanho mínimo (80px web / 25mm impresso) ou acima?
- [ ] Proporção 1.31:1 preservada?
- [ ] Só cores zinc/neutras na decoração?
- [ ] Cores funcionais (verde, vermelho, raridade) usadas apenas como sinal, nunca decoração?
- [ ] Tipografia em Hanken Grotesk?
- [ ] Componentes vindos do shadcn, não recriados manualmente?
- [ ] Assinatura "Hotlist — Diecast Club" com travessão?
- [ ] Nenhuma marca de diecast favorecida no texto?
- [ ] Disclaimer das marcas registradas presente se material for público?
